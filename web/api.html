HTTP API Reference
REST endpoints and WebSocket commands for the Optacon ESP32-S3 web server. For the MicroPython REPL API, see Python API Docs.
/api/device /api/ota /api/wifi /api/music /api/tts /api/exec WebSocket
/api/device Device Status & Diagnostics
GET/api/device/status

Returns firmware version, file version, update URL, and OTA status.

{
  "firmware_version": "0.3.0",
  "files_version": "0.3.0",
  "update_url": "https://example.com/ota",
  "diagnostics_url": ""
}
GET/api/device/log?file=boot

Read a log file. Valid names: boot, boot.prev, check, update.

{
  "log": "Boot OK\nOTA auto-check: up to date"
}
POST/api/device/diagnostics

Send device logs to the configured diagnostics URL.

// Success:  { "status": "ok" }
// Failure:  { "error": "Failed to send diagnostics" }  // 500
/api/ota Over-the-Air Updates
GET/api/ota/config

Get OTA configuration.

{
  "update_url": "https://example.com/ota",
  "auto_check": true,
  "diagnostics_url": ""
}
PUT/api/ota/config

Update OTA configuration. Only included fields are changed.

FieldTypeDescription
update_urlstringURL of the OTA update server
auto_checkboolCheck for updates on boot
POST/api/ota/check

Check for available updates from the configured update server.

{
  "firmware_available": true,
  "latest_firmware": "0.4.0",
  "files_available": false,
  "latest_files": "0.3.0",
  "current_firmware": "0.3.0",
  "current_files": "0.3.0",
  "manifest": { ... }
}
POST/api/ota/update/firmware

Download and install firmware from the update server. Device reboots on success.

FieldTypeDescription
versionstringVersion to install (required)
manifestobjectManifest from /api/ota/check (required)
POST/api/ota/update/files

Download and install file updates from the update server.

FieldTypeDescription
versionstringVersion to install (required)
manifestobjectManifest from /api/ota/check (required)
POST/api/ota/upload

Upload a file or firmware binary directly. Send raw binary body.

Query ParamTypeDescription
filenamestringIf set, uploads as a file. If omitted, uploads as firmware (triggers reboot).
// File upload:   { "status": "ok", "path": "/test.py" }
// FW upload:    { "status": "ok", "message": "Firmware uploaded. Rebooting..." }
/api/wifi WiFi Configuration
GET/api/wifi/status

Returns current WiFi connection status.

{
  "mode": "sta",
  "ssid": "MyNetwork",
  "ip": "192.168.1.42",
  "hostname": "esp-optacon.local"
}
PUT/api/wifi/config

Save WiFi credentials and reconnect. Returns 400 if SSID is empty.

FieldTypeDescription
ssidstringNetwork name (required)
passwordstringNetwork password
{ "msg": "WiFi reconnected", "ip": "192.168.1.42" }
/api/music Sheet Music Playback
GET/api/music/songs

List built-in songs with their note data.

{
  "songs": [
    { "name": "super_mario", "notes": "E5:1 E5:1 ...", "bpm": 400, "gain": 75 }
  ]
}
POST/api/music/play

Play a sequence of notes. Blocks until playback completes. Returns 400 if notes empty or beat value invalid.

FieldTypeDescription
notesstringSpace-separated notes, e.g. "C4:1 E4:1 G4:2" (required)
bpmintBeats per minute (default: 120)
waveformstring"sine", "triangle", or "square" (default: "sine")
gainint25, 50, 75, or 100 Vpp (default: 75)
{ "msg": "music complete" }
POST/api/music/play_song

Play a built-in song by name. Returns 404 if song unknown.

FieldTypeDescription
namestringSong name from /api/music/songs (required)
waveformstring"sine", "triangle", or "square" (default: "sine")
{ "msg": "music complete" }
/api/tts Text-to-Speech
POST/api/tts/say

Speak text using the SAM speech synthesizer. Blocks until speech completes. Returns 400 if text empty.

FieldTypeDescription
textstringText to speak (required)
speedintSpeech speed (default: 72)
pitchintVoice pitch (default: 64)
mouthintMouth shape parameter (default: 128)
throatintThroat shape parameter (default: 128)
{ "msg": "speech complete" }
/api/exec Remote Code Execution
POST/api/exec

Evaluate a Python expression or execute a statement. Tries eval() first, falls back to exec() on SyntaxError. Exceptions are returned in the output.

FieldTypeDescription
codestringPython code to execute
// Expression:  { "output": "4\n" }
// Statement:   { "output": "hello world\n" }
// Error:       { "output": "Traceback ...\nNameError: ..." }
WebSocket /ws — Real-time Hardware Control

Connect to ws://<host>/ws. On connect, the server sends an initial status message. Send JSON commands; receive JSON status or error responses.

Commands
get_status

Request current device status. Always returns full status dict.

// Send:     { "cmd": "get_status" }
// Receive:  { "running": false, "mode": null, "frequency": 0, "gain": 100,
               "fullwave": false, "waveform": "sine", "polarity": false,
               "pins": [0,0,...,0], "amplitude": 100, ... }
start

Start output. Must configure frequency first.

FieldTypeDescription
gainint25, 50, 75, or 100 Vpp (default: 100)
stop

Stop output and put DRV2665 in standby.

set_frequency_analog

Configure analog DDS mode. If already running with same fullwave/dead_time/phase_advance, updates live without restart.

FieldTypeDescription
hzint0 – 1000 Hz (default: 250)
amplitudeint0 – 100% (default: 100)
waveformstring"sine", "triangle", "square", or "dc" (default: "sine")
fullwaveboolEnable fullwave polarity toggle (default: false)
dead_timeintISR ticks after zero-crossing (default: 0)
phase_advanceintISR ticks before zero-crossing (default: 0)
set_frequency_digital

Configure digital FIFO mode (8 kHz sample rate).

FieldTypeDescription
hzint1 – 500 Hz (default: 100)
fullwaveboolEnable fullwave (default: false)
waveformstring"sine", "triangle", or "square" (default: "sine")
set_pin

Set a single HV509 shift register output pin.

FieldTypeDescription
pinint0 – 19
valueint0 or 1
set_all

Set all 20 shift register pins to the same value.

FieldTypeDescription
valuebooltrue = all on, false = all off
set_polarity

Manually set the polarity output (GPIO 12/13).

FieldTypeDescription
valuebooltrue or false
Connection Example
// JavaScript
var ws = new WebSocket('ws://' + location.host + '/ws');
ws.onmessage = function(e) {
  var status = JSON.parse(e.data);
  console.log(status.running, status.frequency);
};
// Start output at 250 Hz
ws.send(JSON.stringify({cmd:'set_frequency_analog', hz:250}));
ws.send(JSON.stringify({cmd:'start', gain:100}));
